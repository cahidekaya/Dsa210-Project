# Dsa210-Project
The Effect of Time Delay and Platform Triggers on Impulsive Purchasing Behavior

   For my term project, I’ll evaluate how online shopping environment affects purchasing behavior, especially if a decision can be considered impulsive or rational. My goal is to combine two aspects: algorithmic triggers ( such as discounts, recommendations, or urgency signals) and temporal action ( time delay before buying).

   I am going to use publicly accessible e-commerce dataset (the Online Retail dataset of Kaggle as an example), which includes transaction-level information contains timestamps, product’s details, and consumer actions. The dataset includes a few thousand transactions over various customers and products, allowing the analysis of purchasing patterns over time.The dataset will be downloaded from Kaggle and processed using Python.

  As public datasets need to be enriched, I am going to create extra features from the basic data. These contain time-based properties (like buying hour, weekday/ weekend, and time among shopping), behavioral characteristics ( like basket size and purchase frequency), and proxy indicators for algorithmic triggers ( discounted goods or quick buying habits as a proxy for urgency as an example).

   In the analysis part, I will first explore the data using EDA to understand general patterns. Then, I will apply hypothesis testing to see whether shorter decision times and certain behaviors are linked to more impulsive purchases. Finally, I will use machine learning methods such as logistic regression and decision trees to classify purchases as impulsive or more rational.
I find this project interesting because it combines consumer behavior with machine learning in a practical setting. One limitation is that impulsiveness is not directly given in the dataset, so I will approximate it using the features I create.
